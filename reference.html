Roles

You and the agents

Docket runs the ticket. You make the calls it is not allowed to make for you. Here is the whole division of labour in one look: the left is yours, the right runs on its own.

You decide

  • Ratify the drafted context — nothing proceeds until you sign off
  • Answer the spec agent's questions — when a ticket is ambiguous
  • Approve or refuse widening the blast radius
  • Decide when to roll back, and to which checkpoint
  • Approve the final merge into real git
  • Act on the retro's findings

Runs without you

  • Reading the ticket, mapping the repo, drafting the plan
  • Saving the original state and a checkpoint per task
  • Writing code inside the agreed boundary
  • Freezing tests, running units, mutation, security, QA
  • Blind peer review of the diff
  • Recording every step to the ledger

Six decisions are yours. Everything else is the machine's — and the one that matters most to you, rolling back, is deliberately not automated: a failed gate or a person chooses to restore, the checkpointer never silently reverts your work.

Commands

Everything you can run

All from inside docket/. Type to filter as the list grows. manual for now marks commands the agent will call for you once the developer loop exists.

Setup one-time

python apply_contract.py

Rewrite the dashboard's schema mapping so it matches your ledger.db. Run it in the folder you actually work from.

python apply_checkpoints_schema.py --db ledger.db

Install the checkpoints and rollbacks tables. Additive and idempotent — it never touches runs, gates, or events.

python apply_checkpoints_schema.py --db ledger.db --doctor

Check the checkpoint tables are present and correct, changing nothing.

Dashboard live

python payload_builder.py --db ledger.db --doctor

Health-check the schema mapping. Reports ok / PARTIAL per table so you catch a drifted column before building a report.

python report.py --db ledger.db --out report.html

Build the self-contained static dashboard you can open or send on.

python serve.py --db ledger.db

Serve the live dashboard on localhost, refreshing as the ledger changes.

Checkpoints · create manual for now

python checkpoint.py init --ticket OT-482 --project onetest --root ..\onetest --radius path1 path2

Start versioning a ticket. Saves the original state (commit #0) before any code is touched. The radius is what may change.

python checkpoint.py init --ticket OT-482 --project onetest --root ..\onetest --radius-file plan\blast-radius.json

Same, but take the radius straight from the lead's blast-radius.json (reads may_touch, ignores must_not_touch).

python checkpoint.py save --ticket OT-482 --task task-01 --stage develop --label "declare mainframe block"

Commit one checkpoint for a finished task. One per task, every task restorable — even a no-op.

python checkpoint.py status --ticket OT-482

Show the checkpoints saved so far and which one your tree matches now.

Checkpoints · restore yours to run

python rollback.py --list-tickets

List every ticket that has checkpoints.

python rollback.py --ticket OT-482

List a ticket's checkpoints, then pick one to restore. Shows a preview of exactly what will change first.

python rollback.py --ticket OT-482 --list

List only, restore nothing.

python rollback.py --ticket OT-482 --to-original

Restore to the state before any agent touched the code. New files created since are removed; changed files reverted.

python rollback.py --ticket OT-482 --to task-05

Restore to a specific checkpoint by task id, number, or sha. Add --yes to skip the confirm.

Self-tests verify

python checkpointer.py --self-test

The git core: byte-identical restore, stray-file removal, the verdict.

python checkpoint_store.py --self-test

Installer is non-destructive; ledger mirror stays in sync with git.

python checkpoint.py --self-test

Init, save, blast-radius parsing, re-init safety.

python rollback.py --self-test

List, resolve, restore to original and forward again, ledger record.

Who does what, stage by stage

The pattern the whole system runs on: an agent (or a person) decides, deterministic code enforces. Nothing that can be computed is left to a model.

Agent decides. Code enforces.

The lead names the files that may change; a hook refuses any edit outside them. The checkpointer restores a tree; git — not a model — proves it is byte-identical. Sequencing, budgets, and gate order belong to the governor, a state machine that cannot rationalise. You are kept in exactly the places where judgement is irreversible: ratifying context, widening scope, rolling back, merging.
